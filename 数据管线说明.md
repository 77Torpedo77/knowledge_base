### Bronze Layer 节点 Schema

**节点类型**：

| Label | 主键 | 说明 |
|-------|------|------|
| `Paper` | `paper_id` | 论文 |
| `Section` | 无 | 论文章节 |
| `TaskMention` | `name` | 研究任务 |
| `MethodMention` | `name` | 提出的方法 |
| `DatasetMention` | `name` | 使用的数据集 |
| `MetricMention` | `name` | 评价指标 |
| `BaselineMention` | `name` | 对比基线 |
| `FlawMention` | `name` | 指出的已有缺陷 |
| `LimitationMention` | `name` | 自述局限性 |

**关系类型**：

| 关系 | 起点 → 终点 |
|------|-------------|
| `HAS_SECTION` | Paper → Section |
| `ADDRESSES_TASK` | Paper → TaskMention |
| `PROPOSES_METHOD` | Paper → MethodMention |
| `USES_DATASET` | Paper → DatasetMention |
| `USES_METRIC` | Paper → MetricMention |
| `COMPARES_WITH_BASELINE` | Paper → BaselineMention |
| `ADDRESSES_FLAW` | Paper → FlawMention |
| `HAS_LIMITATION` | Paper → LimitationMention |

### ⑤ 导出 Bronze 数据 — `bronze_export.py`

| 项目 | 说明 |
|------|------|
| 输入 | Neo4j Bronze Layer（读取所有 *Mention 节点） |
| 输出 | `silver_input.json`（按 Label 分类的扁平 JSON） |
| 功能 | 将 Bronze Mention 按 SAME_AS 连通分量分组，每组取代表名，输出为 LLM 友好格式 |

```bash
python bronze_export.py --output silver_input.json
```

**注意**：在当前状态（只有 Bronze，没有 SAME_AS 关系），导出的就是原始 Mention 数据，不涉及归并。

### ⑥ Silver 层归并 — `silver_builder.py`

| 项目 | 说明 |
|------|------|
| 输入 | `silver_input.json` |
| 输出 | Neo4j `SAME_AS` 关系 + `silver_data.json` + `silver_merge_review.txt` |
| 功能 | 保守确定性规则归并等价 Mention |

**当前规则**：

- **Rule 1**：大小写、空格、连接符、下划线归一化后完全相同 → 合并
- **不再使用 aliases 直接生成 `SAME_AS`**：aliases 仅作为实体属性保留，避免把局部简称、泛称或相关变体误判为全局等价实体。
- **泛称名称保护**：`self-collected dataset`、`our own dataset` 等描述性泛称不作为跨论文 `SAME_AS` 证据。

**已修复的 Bug**：

| Bug | 修复内容 | 日期 |
|-----|---------|------|
| `+` 被当作标点去除 | `normalize()` 现在只去除空白、连接符、下划线，保留 `+`（如 ScanNet++ 不再误合并为 ScanNet） | 2026-05-14 |
| 中文名归一化为空串 | `normalize()` 现在保留非 ASCII 语义字符，不会把纯中文名归一化为空串 | 2026-05-14 |
| 语义符号被误删 | `normalize()` 保留 `*`、`#`、希腊字母、比较符等可能改变语义的符号，避免 `DSO*`→`DSO`、`σ-DVO`→`DVO` 这类误合并 | 2026-06-15 |
| 泛称数据集误合并 | `self-collected dataset`、`our own dataset` 等描述性泛称跳过自动 `SAME_AS` | 2026-06-15 |

```bash
python silver_builder.py --dry-run                                          # 预览
python silver_builder.py --export D:/tools/knowledge_base/silver_data.json  # 执行 + 导出
```

**审查文件**：`silver_merge_review.txt` 列出所有合并详情，标记 `⚠ SUSPICIOUS` 的需人工确认。

### ⑦ Gold 层本体构建 — `gold_ontology_builder.py`

| 项目 | 说明 |
|------|------|
| 输入 | `silver_input.json` |
| 输出 | `gold_output/{Label}_ontology.json` + `{Label}_tree.txt` |
| 功能 | 将每类 Mention 喂给 DeepSeek，让 LLM 构建层级分类本体树 |
| 模型 | `deepseek-v4-flash`，启用 reasoning (thinking) 模式 |
| 分批 | 每批最多 120 个实体，多批次结果直接 extend 合并 |

**LLM 输出的本体结构**：
- `SAME_AS`：真正相同的实体合并为一个 canonical 节点
- `IS_A`：具体算法 → 通用范式的父子层级关系
- 完整性校验：每个 `mention_id` 必须恰好出现一次

**已知潜在问题**：
1. 分批后只是简单 extend 合并，不同批次的同类实体不会被跨批 SAME_AS
2. 没有跨批次本体结构一致性检查
3. SAME_AS vs IS_A 的区分完全依赖 LLM 判断

```bash
python gold_ontology_builder.py                          # 全量
python gold_ontology_builder.py --label BaselineMention  # 单类
```

**需要人工审查** `gold_output/` 下的 JSON 文件后再继续。

### ⑧ Gold 层导入 — `gold_importer.py`

| 项目 | 说明 |
|------|------|
| 输入 | `gold_output/{Label}_ontology.json`（审查后的版本） |
| 输出 | Neo4j `Canonical{Type}` 节点 + `IS_A` 关系 + `RESOLVES_TO` 关系 |
| 功能 | 将 LLM 构建的本体树导入 Neo4j Gold Layer |

```bash
python gold_importer.py --dry-run               # 预览
python gold_importer.py                          # 全量导入
python gold_importer.py --label BaselineMention  # 单类
```

**新增的 Neo4j 节点和关系**：

| Label | 主键 | 说明 |
|-------|------|------|
| `CanonicalTask` | `canonical_id` | 金层规范任务实体 |
| `CanonicalMethod` | `canonical_id` | 金层规范方法实体 |
| `CanonicalDataset` | `canonical_id` | 金层规范数据集实体 |
| `CanonicalMetric` | `canonical_id` | 金层规范指标实体 |
| `CanonicalBaseline` | `canonical_id` | 金层规范基线实体 |
| `CanonicalFlaw` | `canonical_id` | 金层规范缺陷实体 |
| `CanonicalLimitation` | `canonical_id` | 金层规范局限性实体 |

| 新关系 | 起点 → 终点 | 说明 |
|--------|-------------|------|
| `IS_A` | Canonical → Canonical | 层级（子类 → 父类） |
| `RESOLVES_TO` | Mention → Canonical | Bronze → Gold 映射 |
| `SAME_AS` | Mention → Mention | Silver 层等价 |
