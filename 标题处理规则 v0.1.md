下面是一套可以直接交给 agent 实现的 **标题处理规则 v0.1**。

---

# 标题处理规则 v0.1

## 目标

从 `content_list_v2.json` 中识别论文标题、章节标题、子章节标题，并生成带层级关系的 `sections.json`。

输出中必须保留：

```text
raw_title：论文原始标题
normalized_type：标准章节功能类型
level：标题层级
parent_id：父章节 ID
```

---

# 1. 标题识别规则

## 1.1 优先使用 MinerU 的标题标记

如果内容块的类型是：

```text
title
```

或包含类似字段：

```text
text_level
```

则优先认为它是章节标题。

---

## 1.2 用编号辅助判断标题层级

如果标题文本符合以下模式，也可判断为标题：

```text
1 Introduction
2 Related Work
3 Method
3.1 Module A
3.2 Module B
4 Experiments
4.1 Datasets
4.2 Results
```

规则：

```text
1 / 2 / 3          → level = 1
1.1 / 2.3 / 4.2    → level = 2
1.1.1 / 2.3.1      → level = 3
```

示例：

```text
3 Method                  level = 1
3.1 Feature Extraction    level = 2
3.1.1 Encoder Design      level = 3
```

---

## 1.3 无编号标题的层级判断

如果标题没有编号，则按 MinerU 的 `text_level` 判断。

如果没有 `text_level`，则默认：

```text
level = 1
```

除非它出现在某个大章节内部，并且标题较短、明显是子模块名，则可设为：

```text
level = parent_level + 1
```

---

# 2. 标题清洗规则

对标题做轻量清洗：

```text
去掉首尾空格
去掉多余换行
去掉标题前的编号，但保留原始标题 raw_title
```

例如：

```text
raw_title = "3.1 Proposed Module"
clean_title = "Proposed Module"
```

但输出中仍保留：

```json
"raw_title": "3.1 Proposed Module"
```

---

# 3. 标准章节类型 normalized_type

每个标题都要映射到以下类型之一：

```text
abstract
introduction
background_or_related_work
problem_formulation
method
experiment_or_evaluation
dataset
result_or_analysis
discussion
limitation_or_future_work
conclusion
references
appendix
other
```

---

# 4. 标题到 normalized_type 的映射规则

## 4.1 Abstract

标题包含：

```text

```

映射为：

```text
abstract
```

---

## 4.2 Introduction

标题包含：

```text
introduction

```

映射为：

```text

```

## 4.3 Background / Related Work

标题包含：

```text

```

映射为：

```text
background_or_related_work
```

## 4.4 Problem Formulation

标题包含：

```text
problem formulation
problem definition

formulation

```

映射为：

```text
problem_formulation
```

```text
method
```

映射为：

```text
method
```

# 5. 父子章节规则

使用 `level` 建立父子关系。

规则：

```text
level = 1 → parent_id = null
level = 2 → parent_id = 最近的 level 1 section
level = 3 → parent_id = 最近的 level 2 section
```

示例：

```json
{
  "section_id": "paper-S03",
  "raw_title": "3 Method",
  "level": 1,
  "parent_id": null,
  "normalized_type": "method"
}
```

```json
{
  "section_id": "paper-S03-01",
  "raw_title": "3.1 Module A",
  "level": 2,
  "parent_id": "paper-S03",
  "normalized_type": "method"
}
```

---

# 6. 子章节类型继承规则

默认情况下，子章节继承父章节的 `normalized_type`。

例如：

```text
3 Method                  → method
3.1 Module A              → method
3.2 Module B              → method
3.3 Training Strategy     → method
```

但如果子章节标题明显属于其他类型，则覆盖父类型。

例如：

```text
4 Experiments             → experiment_or_evaluation
4.1 Datasets              → dataset
4.2 Baselines             → experiment_or_evaluation
4.3 Results               → result_or_analysis
4.4 Ablation Study        → result_or_analysis
```

---

# 7. 非标题内容归属规则

普通段落、表格、公式、图片说明，应归入最近一个已识别 section。

如果在第一个标题之前出现内容：

```text
如果像摘要 → 放入 abstract
否则 → 放入 front_matter
```

---

# 8. References 处理规则

一旦遇到：

```text
normalized_type = references
```

则后续内容默认丢弃，不进入正文。

但保留一个 references section 元数据即可：

```json
{
  "raw_title": "References",
  "normalized_type": "references",
  "text": ""
}
```

---

# 9. 输出格式

最终输出 `sections.json`：

```json
{
  "paper_id": "citekey",
  "title": "paper title",
  "sections": [
    {
      "section_id": "citekey-S01",
      "order": 1,
      "level": 1,
      "parent_id": null,
      "raw_title": "1 Introduction",
      "clean_title": "Introduction",
      "normalized_type": "introduction",
      "page_start": 1,
      "page_end": 2,
      "text": "..."
    },
    {
      "section_id": "citekey-S03-01",
      "order": 4,
      "level": 2,
      "parent_id": "citekey-S03",
      "raw_title": "3.1 Module A",
      "clean_title": "Module A",
      "normalized_type": "method",
      "page_start": 4,
      "page_end": 5,
      "text": "..."
    }
  ]
}
```

---

最重要的原则是：

```text
保留原始标题 raw_title
识别功能类型 normalized_type
保留层级 level
用 parent_id 表示父子关系
子章节默认继承父章节类型，必要时覆盖
```
